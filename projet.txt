 1. Introduction (Le "Pourquoi")
  "Bonjour à tous. Aujourd'hui, je vous présente ApexGrid.
  Vous connaissez tous l'univers de la Formule 1, un sport où chaque milliseconde compte. Mais comment transformer cette passion pour les
  données en une expérience interactive et compétitive ? C'est le défi que j'ai relevé avec ApexGrid : une plateforme de Fantasy F1 qui ne
  se contente pas de comptabiliser des points, mais qui analyse en profondeur la performance réelle des pilotes et des écuries."

  2. Le Concept (Le "Quoi")
  "ApexGrid est une application web immersive qui permet aux utilisateurs de devenir de véritables directeurs d'écurie.
   - Le cœur de l'expérience : composer sa propre équipe de pilotes à chaque Grand Prix.
   - La particularité : le score ne dépend pas seulement de la position à l'arrivée, mais d'un algorithme propriétaire qui valorise les
     dépassements, les duels entre coéquipiers, et la capacité à sublimer des monoplaces de fond de grille. En somme, c'est une Fantasy F1
     axée sur la performance relative."

  3. L'Architecture Technique (Le "Comment" - Partie Technique)
  "Pour garantir une expérience fluide, même lors des pics de trafic après une course, j'ai opté pour une architecture robuste :
   - Frontend : Une SPA React avec TypeScript et TailwindCSS. L'interface est pensée pour être rapide, dynamique, et surtout réactive.
   - Backend : Un serveur Node.js avec Express. C'est ici que réside la 'magie' : le moteur de scoring.
   - Base de données : Supabase (PostgreSQL), qui permet une gestion sécurisée et performante des données structurées.
   - Communication : API RESTful sécurisée par JWT pour garantir que chaque utilisateur ne modifie que ses propres équipes."

  4. Le Moteur de Scoring (Le Différenciateur)
  "Ce qui rend ApexGrid unique, c'est son moteur de traitement de données.
  Concrètement, comment cela fonctionne ?
   1. Ingestion : Le système importe automatiquement les résultats officiels de la F1.
   2. Traitement Dynamique : Le SyncManager identifie, avant même le calcul, les pilotes d'élite (BigFive) et les opportunités du fond de
      grille (BottomTeams).
   3. ScoringEngine : C'est le cœur logique. Il croise les données de position, les bonus de performance et les facteurs de risque pour
      calculer en quelques secondes le score total pour chaque utilisateur.
   4. Temps Réel : Une fois calculés, les résultats sont poussés vers le frontend via des notifications, garantissant une mise à jour
      immédiate."

  5. Fonctionnalités et Expérience Utilisateur
  "Côté utilisateur, tout est pensé pour la simplicité :
   - Un Dashboard pour suivre les classements en temps réel.
   - Une section Historique intuitive pour gérer ses équipes round après round.
   - Un Wiki F1 intégré pour consulter les statistiques et préparer ses choix stratégiques avec des données concrètes.
   - Enfin, un panel d'administration dédié permet de gérer les utilisateurs et de déclencher les synchronisations de données avec une
     grande flexibilité."

  6. Conclusion
  "Pour conclure, ApexGrid est plus qu'un jeu de pronostics. C'est une plateforme d'analyse de données F1 au service de l'engagement
  utilisateur. C'est la fusion entre la passion du sport automobile et une ingénierie logicielle rigoureuse.
  Merci pour votre attention, je suis maintenant prêt à répondre à vos questions."
